Git is a version checking tool commonly used in the CS industry. It allows companies to maintain a "main" body of code while giving engineers permission to "pull" code from the main branch to tweak. If they do tweak the code, they must then add it back to the main branch in the following manner.

  1. Add: By adding the new file to the "commit" list, the computer then knows to add it to the local main branch. This is done using the command git add [FILE NAME]
  2. Commit: Committing just adds any new files (or any modified files) to the local main branch. It is done with the command git commit -m"[CHANGELOG]". The strength of git comes with the included changelog, where you can detail what you changed in the code (or what the code is). This way, any new commit (whether it be a new body of code or just an edit of existing code) can be easily and quickly categorized. the -m is an important flag as well in the ssh code, but explaining it escapes the scope of this current document.
  3. Push: A push request is a request to push all files on the local main branch to the global main branch (for example, a company that asks you to change something will have a main branch in their repository, and after you change something you would need to add your code back to the main repository.)

You can check the status of your local github branches with the command git status.

As another note, you do need to create some repository URL that connects your initialized git instance on your machine to the repository you are working from.
